Mirador

The viewer provides inspection and comparison functionality for scholars studying large, high-resolution images of manuscripts and other artefacts. User stories constructed from interviews with leading manuscript scholars have inspired the design of the functionality represented in the current generation of mock-ups.

The viewer is meant to provide an inspection environment for resources discovered through an external search and browsing system, such as SearchWorks.

  • Viewer demo

Multiple Repositories

It should be noted that the viewer's interface draws from any remotely served repositories that provide a IIIF-compliant Metadata API, even from separate institutions. This means that scholars can simultaneously examine related artefacts from multiple collections served by any institution in the world.

Easy Implementation

The viewer is easy to implement, following a similar pattern to many jQuery plugins. Implementing developers need only drop in the javascript library and call Mirador({options}), and they have a working multi-up viewer instance. "Options" are specified as a simple JSON string enumerating the repositories from which to draw, the windows to open, and the general application settings.
